# SEO Readiness Checklist

This document confirms the technical SEO implementation for Awais Digital Services.

## 1. Crawlability

- **noindex**: No `noindex` meta tags on any indexable page.
- **robots.txt**: Generated by `src/app/robots.js`. Allows `User-agent: *` with `Allow: /`. Disallows only `/api/`, `/_next/`, `/admin/`.
- **Canonicals**: Set via `metadata.alternates.canonical` on homepage, services, pricing, projects, contact, legal pages, and each project detail page.
- **HTTP headers**: Security headers (CSP, X-Frame-Options, etc.) do not block crawlers. No `X-Robots-Tag` blocking.

## 2. Robots.txt

- **Location**: Served at `/robots.txt` (Next.js generates from `src/app/robots.js`).
- **Content**: `User-agent: *`, `Allow: /`, `Disallow: /api/`, `/_next/`, `/admin/`.
- **Sitemap**: `Sitemap: https://www.awaisdigitalservices.co.uk/sitemap.xml`

## 3. XML Sitemap

- **Location**: `/sitemap.xml` (generated by `src/app/sitemap.js`).
- **Includes**: Homepage, `/contact`, `/services`, `/pricing`, `/projects`, `/cookie-policy`, `/privacy-policy`, `/terms-and-conditions`, and all `/projects/[id]` URLs.
- **Note**: No blog section exists; when added, include `/blog` and blog post URLs in the sitemap.

## 4. Internal Linking

- **Homepage**: Links to Services, Contact (Hero); View Pricing, See Our Projects (Hero); Pricing section and Portfolio link to `/pricing` and `/projects`; Navbar links to Home, Services, Projects, Pricing, Contact.
- **Services page**: Links to Home, Pricing, Our Projects, Contact Us, and WhatsApp.
- **Footer**: Quick Links (Services, Projects, Pricing, Contact), Services list, Contact info.

## 5. Page Metadata

- **Titles**: Unique, keyword-aware titles on all pages (home, services, pricing, projects, contact, legal, project details).
- **Meta descriptions**: Unique descriptions on all pages.
- **Canonical**: Set on all indexable pages.
- **Open Graph & Twitter**: Implemented on all main and legal pages.

## 6. Heading Structure

- **One H1 per page**: Home (Hero), Services, Pricing, Projects, Contact, Privacy, Cookie, Terms, and each project detail page use a single H1.
- **H2/H3**: Used for major sections and subsections consistently.

## 7. Structured Data (JSON-LD)

- **Organization**: In root layout `<head>` (name, url, logo, email, telephone, address, areaServed, contactPoint, sameAs).
- **LocalBusiness**: In root layout (extends organization with geo, serviceArea, hasOfferCatalog).
- **WebSite**: In root layout (name, url, publisher, potentialAction SearchAction).
- **BreadcrumbList**: Rendered by `BreadcrumbSchema` in layout (path-based breadcrumbs for all pages).

## 8. Indexable Pages

All of the following are intended to be indexable (no noindex, linked from sitemap and internal links):

- Homepage (`/`)
- Services (`/services`)
- Pricing (`/pricing`)
- Projects (`/projects`)
- Project detail pages (`/projects/[id]`)
- Contact (`/contact`)
- Privacy Policy (`/privacy-policy`)
- Cookie Policy (`/cookie-policy`)
- Terms and Conditions (`/terms-and-conditions`)

## 9. URL Structure

- Clean URLs: `/services`, `/projects`, `/contact`, `/pricing`, `/privacy-policy`, `/cookie-policy`, `/terms-and-conditions`, `/projects/[slug]`.
- `trailingSlash: false` in `next.config.mjs` for consistent non-trailing-slash URLs.

## 10. Verification

- **Status 200**: All above routes are valid Next.js pages and return 200 when built and run.
- **Sitemap**: Contains all listed indexable URLs.
- **Robots**: Allows crawling of all public pages; only technical paths disallowed.
